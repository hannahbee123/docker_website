Setup a Web Server Using Docker

First, download Docker desktop. Then create a Docker Hub account. After installing Docker desktop and logging into your Docker Hub account, open a CLI shell (such as PowerShell for PC or Terminal for Mac) and run “docker run hello-world” to test your installation.

Docker Hub allows you to pull pre-built images onto your local computer. To pull an Ubuntu image from Docker Hub, run “docker pull ubuntu” in your CLI shell.

After you pull the Ubuntu image from docker, you should see it listed in your Docker images. Run “docker images” to see images on your local computer.

Next, run the Ubuntu image to create and log in to a Docker container by using the command “docker run -it ubuntu:latest /bin/bash”. This brings you to an interactive shell.

The characters following ‘root@’ in the shell represent the container ID. Next, update the system using the command “apt-get update”. When the updates have finished installing, it’s time to install Apache, a Web server, in your container. To do this, run “apt-get install apache2”. After Apache has finished installing, run “apt-get install vim” to install Vim, a text editor where you can use HTML to create websites. Once Vim has finished installing, exit the interactive container shell by using the command “exit”. Now it’s time to commit the container and create a new docker image. Run “docker ps -a” to check the container.

Now, create a new Docker image of Ubuntu with Apache by running ““docker commit [CONTAINER ID] [NAME OF NEW IMAGE]”.

In the example above, ‘c123ac8864d5’ is the container ID and ‘ubuntu-my_apache2’ is the name of the new image. Run “docker images” to make sure your image has been saved.

You now have a container running Ubuntu and Apache on your local computer.
